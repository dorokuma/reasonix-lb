### P2 — 中等风险

#### #1 http.Server 缺少超时配置
- **文件**: `main.go:28`
- **代码**: `srv := &http.Server{Addr: cfg.Listen, Handler: NewProxyHandler(pool)}`
- **问题**: `ReadTimeout`、`WriteTimeout`、`ReadHeaderTimeout` 均未设置。慢客户端可以极低速率发送数据，保持连接长期打开，导致 goroutine 和文件描述符泄漏。
- **修复**: 补充配置项并在 `http.Server` 中设置：
  ```go
  &http.Server{
      Addr:              cfg.Listen,
      Handler:           NewProxyHandler(pool),
      ReadHeaderTimeout: 10 * time.Second,
      ReadTimeout:       30 * time.Second,
      WriteTimeout:      60 * time.Second,
  }
  ```

#### #2 请求体无大小限制
- **文件**: `proxy.go:45`
- **代码**: `bodyBytes, err := io.ReadAll(r.Body)`
- **问题**: 整个请求体被读入内存，无上限。攻击者可发送数 GB 请求体导致 OOM。
- **修复**: 使用 `http.MaxBytesReader` 限制读取：
  ```go
  r.Body = http.MaxBytesReader(w, r.Body, 32<<20) // 32 MiB
  bodyBytes, err := io.ReadAll(r.Body)
  ```

#### #3 上游请求未传播客户端 context
- **文件**: `proxy.go:59`（proxyChat）、`proxy.go:120`（proxyModels）
- **代码**: `http.NewRequest("POST", ..., body)` / `http.NewRequest("GET", ..., nil)`
- **问题**: 使用 `context.Background()`，客户端断开连接后上游请求继续执行。对于 LLM 代理场景，这可能浪费上游配额或产生重复 completion（客户端超时重试时）。
- **修复**: 改用 `NewRequestWithContext`：
  ```go
  req, err := http.NewRequestWithContext(r.Context(), "POST", ...)
  // 以及
  req, err := http.NewRequestWithContext(r.Context(), "GET", ...)
  ```

### P3 — 低风险

#### #4 JSON 错误响应 Content-Type 为 text/plain
- **文件**: `proxy.go:47,55,109,117,166`
- **代码**: `http.Error(w, JSON字符串, statusCode)`
- **问题**: `http.Error` 内部设置 `Content-Type: text/plain; charset=utf-8`，但 body 是 JSON。严格客户端可能拒绝处理。
- **修复**: 手动设置 Content-Type 并写入 body，或改用 `json.NewEncoder(w).Encode(...)`：
  ```go
  w.Header().Set("Content-Type", "application/json")
  w.WriteHeader(503)
  json.NewEncoder(w).Encode(map[string]any{...})
  ```

#### #5 配置文件路径硬编码
- **文件**: `main.go:14`
- **代码**: `LoadConfig("config.yaml")`
- **问题**: 不支持通过环境变量或命令行参数覆盖路径，部署灵活性受限。
- **修复**: 支持环境变量 `CONFIG_PATH`：
  ```go
  cfgPath := os.Getenv("CONFIG_PATH")
  if cfgPath == "" {
      cfgPath = "config.yaml"
  }
  cfg, err := LoadConfig(cfgPath)
  ```

### P4 — 信息性

#### #6 JSON 错误体尾部多换行符
- **文件**: `proxy.go:47,55,109,117,166`
- **问题**: `http.Error` 使用 `fmt.Fprintln` 写入 body，尾部追加 `\n`。JSON 解析器通常能接受，但不规范。

- **问题**: `go test` 报告 `[no test files]`，关键逻辑（选择、耗尽标记、重试边界）没有单元测试覆盖。
